linkModifying the Skybox and scene lighting

Open the scene which you wish to edit The skybox can then be changed by using the menu at the top left of the Unity window Navigate to Window > Rendering > Lighting Once the Lighting Window has been opened switch to the environment tab and you will see the Skybox material and settings.

linkModifying Materials

It is good practice to duplicate the material you are editing and then assign the duplicated material to the object, this duplicate can then be edited freely as modifications to the original material will appear on every object that has that material assigned to it.

You can assign a material by finding it in the project files and then dragging it onto the object you want that material applied to.

Any visible object within a scene can have its color or texture changed by clicking on it in the Hierarchy and finding the material component in the inspector on the right hand side of the screen.

linkWarnings and Errors

Within the console tab at the bottom of the Unity window you will get printouts of various types.

The white circle is used for standard print and debug messages. The yellow triangle is used for warnings. The red octagon is used for errors.

Throughout the project many of these messages to prompt the user when certain events take place, warnings have been used when something that may not have been the user's intention has taken place. Errors are used when something that breaks the program or interferes with the accurate collection of trial data occurs.

For example, when setting up interpolations one or more trials within the sequence may not have the required OOI, this is not necessarily wrong and may be intentional so only a warning is used and not an error.

Whereas, when a critical component such as the UDP Server which receives packets from DLC-live cannot be found an error message is presented as this will make the operation of closed loop and the collection of data unviable.

Warning and error messages that have been written into the CAVE project will start with the text “Warning: “ and “Error: “ respectively, warnings and errors that are native to Unity will not start with this text.

Clicking on any output within the console will bring up details about the code that was executed prior to the error, clicking on the highlighted text will take you to the line of code being referenced and help you to understand what may have caused the error to occur.

Double clicking on the console printout will take you to the line of code where the error was printed / occured.

linkMiscellaneous

Renaming scenes will cause all scriptable objects that reference that scene to lose the reference, they will then switch to the default value which will be the MainMenu scene. If you wish to restore the original reference it must be done manually.

Do not use the ‘/’ character in the names of scriptable objects, animal number text-field, or any other text fields that end up being used to generate file names as it will cause a folder to be created and possibly ruin scripts that make use of file paths. The (A-Z, a-z and 0-9) characters are the safest to use when naming

When play is ended it is common for the following message to be printed in the console System.Threading.ThreadAbortException from UDPServer This is fine and occurs due to the UDPSocket being terminated when the UDPServer script is destroyed.

Screen / Display Setup with Ubuntu Use Ubuntu display settings in addition to the Nvidia Settings when setting up the screen configuration. For Unity to set up the Stimulus Screens they must be given the left most position within the Ubuntu display setting, as this allows them to be auto targeted for the creation of editor windows (this is what breaks the add component button and causes most windows to open on the wrong screen).

If you cannot see your stimuli on stimulus screens, especially after pulling update to the White computer, go MainMenu -> User Interface Tick a box for “Is activating Stimuli screen” in In UI_Manager

Then, Apply to Prefab ‘User Interface’ via Modified Components in a pulldown menu for UI_Manager(script) and save the MainMenu.

In order to run both DLC-live and Unity, Unity must enter play mode and Stimulus Screens must be initialised first, then dlc-live can be activated, not sure why but Unity crashes when trying to initialise stimulus screens if dlc-live is already active.

guvcview can be used to set the camera settings, this will then be used by the camera when run via dlc-live, if dlc-live only shows a grey video that changes to black and white when objects are shown. reset the camera settings to hardware default in guvcview.
